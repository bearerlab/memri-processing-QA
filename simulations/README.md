# simulations/

This subdirectory contains code used to generate synthetic noise-only and simulation datasets, process them through the MEMRI pipeline, perform statistical analyses, and produce corresponding figures. These simulations were used to validate QA metrics and characterize expected signal and noise behavior.

## Scripts

### R Scripts

| Script | Description |
|---|---|
| `simulate_images.R` | Generates noise-only and simulated MEMRI datasets with user-specified parameters for signal level, noise distribution, and group structure. |
| `FPR_FNR_Analysis.R` | Produces publication-quality figures summarizing simulation results and QA metric performance across simulated conditions. |

### Bash Scripts

| Script | Description |
|---|---|
| `ExtractConfusion.sh` | Counts the number of total voxels, the total number predicted positive voxels (from SPM t-tests below), the total number of actual positive voxels, and the total number of true positive voxels. |

### MATLAB / SPM Batch Scripts

| Script | Description |
|---|---|
| `SPMBatch_SmoothFiles_job.m` | SPM12 batch script for smoothing simulated volumes by 3 different kernels. |
| `SPMBatch_Ttest_Negatives_s*_job.m` | SPM12 batch script for paired t-test simulated 'noise-only'/'false-negative' volumes. |
| `SPMBatch_Ttest_Positives_s*_job.m` | SPM12 batch script for paired t-test simulated volumes with positive signal embedded in noise. |

## Notes

- Simulated data generated by `simulate_images.R` are used as inputs to the MATLAB/SPM batch scripts. Note that this script generates the GridMasks folder and NIfTI volumes inside.  `ExtractConfusion.sh` extracts false and true positive voxel counts and pushes data to `ConfusionData.csv`, which was used to generate the full `ConfusionData.xlsx`. The resultant Excel file was processed and graphed in `FPR_FNR_Analysis.R`. 
- MATLAB scripts require SPM12 (see [SPM12 documentation](https://www.fil.ion.ucl.ac.uk/spm/software/spm12/)).
